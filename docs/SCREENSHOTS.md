## Capture quality

- Use the same browser, viewport, and zoom level throughout.
- Capture focused content without browser developer tools.
- Ensure buttons, headings, and important values are not cut off.
- Avoid empty panels unless the screenshot demonstrates an empty state.
- Check spelling and visible timestamps before capturing.
- Verify severity colors remain legible and text has adequate contrast.

## Safety check

Before every capture:

- remove API keys, tokens, email addresses, and personal directories
- remove customer, employer, and production-system information
- use only `198.51.100.0/24` and `203.0.113.0/24` documentation addresses
- use the fictional host and account included in the guided demo
- close unrelated tabs and hide bookmarks or notifications
- inspect PDF metadata and filenames for personal information
