# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability, please report it privately instead of creating a public issue.

Include:

* Description of the issue
* Steps to reproduce
* Potential impact
* Suggested mitigation (if available)

Please avoid publicly disclosing vulnerabilities until they have been reviewed and addressed.

## Sensitive Data

ThreatLens is intended for defensive cybersecurity analysis.

Users should submit only sanitized or fictional security-event data.

Never submit:

* Passwords
* Authentication tokens
* Private keys
* Personally identifiable information
* Customer secrets
